360 Chat-Controlled Camera Setup Guide

Guide v4.2 · July 16, 2026

Turn an Insta360 X5 (or any equirectangular 360 camera) into a camera your Twitch chat can drive. Viewers pan, tilt, zoom, and roll the view with !cam and !zoom, trigger a full !spin or !barrelroll, and every new follower spins the camera automatically.

Part 1

Prerequisites

Everything in this guide is embedded below — both code files have copy buttons. You'll paste the shader into a text file for OBS and the C# script directly into Streamer.bot.

  1. OBS Studio 28+ with the WebSocket server enabled: Tools → WebSocket Server Settings → Enable, port 4455. Note the password if you set one.
  2. obs-shaderfilter plugin — download from the plugin's GitHub releases page for your OBS version and OS. Restart OBS after installing.
  3. Streamer.bot connected to both Twitch and OBS (Stream Apps → OBS Studio → add connection on port 4455).
  4. Your 360 camera feeding OBS in equirectangular mode. The Insta360 X5 in webcam mode outputs 2:1 panoramic at 2880×1440.
Part 2

OBS: Capture the camera

  1. Add a Video Capture Device source for the camera. Name it something exact and memorable, e.g. Insta360 X5 — you'll need this name character-for-character in the script later.
  2. In the source properties, uncheck "Use Preset" and manually set the resolution to 2880×1440 at 30 FPS. If OBS defaults to 1920×1080 or 720p here, your final image quality tanks — this is the #1 quality complaint.
  3. Confirm the raw feed looks like a stretched-out unwrapped panorama. That's equirectangular — correct. If you see a normal flat view, the camera is in reframe/single-lens mode instead.
Part 3

Install the reframe shader

  1. Copy the shader below and save it as mushbox-360-reframe.shader anywhere on your PC (Notepad is fine — just make sure the extension isn't secretly .txt).
  2. In OBS: right-click the camera source → Filters → + → User-defined shader. Keep the default filter name User-defined shader (or note what you name it — the script needs it exactly).
  3. In the filter, check Load shader text from file and browse to the file you saved.
  4. You should now see a flat, normal-looking view with sliders: Yaw, Pitch, Roll, Field of view, Output aspect. Set Output aspect to 1.7778 for 16:9. FOV around 95–115 looks natural.
  5. Fix the frame: right-click the source → Transform → Edit Transform → Bounding Box Type Stretch to bounds, size 1920×1080 (or your canvas size).
Sanity check: drag the Yaw slider by hand. The view should pan smoothly through the full 360, including cleanly across the seam behind the camera.
mushbox-360-reframe.shader (v2.2 — seam + pitch-direction fixes included) 
// ============================================================
// Mushbox 360 Reframe Shader (for obs-shaderfilter) — v2.2
// Converts equirectangular 360 video (e.g. Insta360 X5, 2:1)
// into a flat rectilinear view with adjustable yaw / pitch /
// roll / FOV / output aspect.
//
// v2.2: fixes inverted pitch direction (up now looks up)
// v2.1: fixes the vertical seam visible when facing the rear
// (yaw 180) by manually blending across the texture edge.
// Uses only the plugin's built-in sampler — no sampler_state
// block, which obs-shaderfilter's parser rejects.
//
// IMPORTANT: The uniform names below (yaw, pitch, roll, fov,
// output_aspect) are what the Streamer.bot script writes to.
// If you rename them, rename them in the C# script too.
// ============================================================

uniform float yaw<
    string label = "Yaw (degrees)";
    string widget_type = "slider";
    float minimum = -360.0;
    float maximum = 360.0;
    float step = 0.1;
> = 0.0;

uniform float pitch<
    string label = "Pitch (degrees)";
    string widget_type = "slider";
    float minimum = -90.0;
    float maximum = 90.0;
    float step = 0.1;
> = 0.0;

uniform float roll<
    string label = "Roll (degrees)";
    string widget_type = "slider";
    float minimum = -180.0;
    float maximum = 180.0;
    float step = 0.1;
> = 0.0;

uniform float fov<
    string label = "Field of view (degrees)";
    string widget_type = "slider";
    float minimum = 30.0;
    float maximum = 150.0;
    float step = 0.5;
> = 95.0;

uniform float output_aspect<
    string label = "Output aspect (width/height, 1.7778 = 16:9)";
    string widget_type = "slider";
    float minimum = 0.5;
    float maximum = 4.0;
    float step = 0.0001;
> = 1.7778;

#define PI 3.14159265358979

float4 mainImage(VertData v_in) : TARGET
{
    // --- Build a camera ray for this output pixel ---
    float fovRad  = radians(clamp(fov, 30.0, 150.0));
    float halfTan = tan(fovRad * 0.5);

    // Normalized device coords: x right, y up, centered at 0
    float ndcX = (v_in.uv.x - 0.5) * 2.0;
    float ndcY = (0.5 - v_in.uv.y) * 2.0;

    float3 ray = normalize(float3(
        ndcX * halfTan,
        ndcY * halfTan / output_aspect,
        1.0
    ));

    // --- Roll (around forward/z axis) ---
    float cr = cos(radians(roll));
    float sr = sin(radians(roll));
    ray = float3(cr * ray.x - sr * ray.y,
                 sr * ray.x + cr * ray.y,
                 ray.z);

    // --- Pitch (around x axis) ---
    float cp = cos(radians(pitch));
    float sp = -sin(radians(pitch));   // v2.2: negated so positive pitch looks UP
    ray = float3(ray.x,
                 cp * ray.y - sp * ray.z,
                 sp * ray.y + cp * ray.z);

    // --- Yaw (around vertical/y axis) ---
    float cy = cos(radians(yaw));
    float sy = sin(radians(yaw));
    ray = float3( cy * ray.x + sy * ray.z,
                  ray.y,
                 -sy * ray.x + cy * ray.z);

    // --- Ray direction -> equirectangular UV ---
    float lon = atan2(ray.x, ray.z);              // -PI..PI
    float lat = asin(clamp(ray.y, -1.0, 1.0));    // -PI/2..PI/2

    float u = frac(lon / (2.0 * PI) + 0.5);
    float v = 0.5 - lat / PI;

    // --- Seam fix: manually blend across the texture edge ---
    // Bilinear filtering clamps at the left/right edges, which
    // draws a thin line when the view crosses the rear seam.
    // Inside a one-texel band at each edge, blend the centers of
    // the first and last pixel columns instead.
    float w = max(uv_pixel_interval.x, 0.0001); // one texel width

    if (u < w || u > 1.0 - w)
    {
        float t = frac(u + w) / (2.0 * w); // 0 at right edge -> 1 at left edge
        float4 cRight = image.Sample(textureSampler, float2(1.0 - 0.5 * w, v));
        float4 cLeft  = image.Sample(textureSampler, float2(0.5 * w, v));
        return lerp(cRight, cLeft, t);
    }

    return image.Sample(textureSampler, float2(u, v));
}
Part 4

Streamer.bot: the script

  1. Actions tab → right-click → Add → name it Cam360 Control.
  2. In its Sub-Actions pane: right-click → Core → C# → Execute C# Code.
  3. Delete all the placeholder code and paste in the entire script below. Don't paste it inside the existing template — replace everything.
  4. Edit the config block at the top: SourceName and FilterName must match OBS exactly (case, spaces, everything). ObsConnection is 0 if OBS is your first/only connection in Stream Apps.
  5. Click the References tab and confirm these exist (add via "Add from Assembly Name" if missing): mscorlib.dll, System.dll, System.Core.dll.
  6. Compile → look for "Compiled Successfully" → Save and Compile → OK.
Heads up: a missing System.Core.dll reference is the most common compile failure. If you see errors about dynamic binding or missing references, that's the fix.
Cam360Control.cs v4.2 (pan / tilt / zoom / roll) — full drop-in replacement 
using System;
using System.Globalization;
using System.Threading;

// ============================================================
// Mushbox Cam360 Control — v4.2 (adds barrel roll + roll angle)
// Built on v4.1. Presets (front/back/left/right) now re-level
// BOTH pitch and roll; "level" fully levels the camera.
// Zoom still holds across pans until changed or reset.
//
// camAction / !cam values:
//   front|back|left|right  -> pan to heading, re-level pitch + roll
//   up|down|level          -> pitch presets (level also zeroes roll)
//   tilt <-90..90>         -> pitch to exact angle
//   roll <-180..180>       -> dutch angle (e.g. roll 15)
//   barrelroll             -> full 360 roll, ends where it started
//   <degrees>              -> yaw only
//   spin                   -> full 360 yaw spin (holds pitch + zoom)
//   zoomin|zoomout         -> step FOV down/up by ZoomStep
//   zoom <fov>             -> exact FOV (lower = punched in)
//   zoomreset              -> back to DefaultFov
//
// Commands: !cam <value>, !spin, !zoom <...>, !barrelroll
//
// Clamping: chat zoom is clamped to ChatFovMin..ChatFovMax;
// deck wrappers (camAction) get the full DeckFovMin..DeckFovMax.
// ============================================================

public class CPHInline
{
    // ======= EDIT THESE TO MATCH YOUR OBS SETUP =======
    private const string SourceName = "Insta360 X5";
    private const string FilterName = "User-defined shader";
    private const int    ObsConnection = 0;

    // ======= ANIMATION TUNING =======
    private const int    StepMs              = 25;
    private const double PanDurationSec      = 1.5;
    private const double SpinDurationSec     = 4.0;
    private const double ZoomDurationSec     = 1.0;
    private const double BarrelRollDurationSec = 2.5;
    private const bool   ChatFeedback        = true;

    // ======= ZOOM TUNING =======
    private const double DefaultFov  = 115.0;  // your "home" FOV — wider = sharper (see Image Quality section)
    private const double ZoomStep    = 15.0;   // FOV degrees per zoomin/zoomout step
    private const double DeckFovMin  = 30.0;   // full shader range (deck / camAction path)
    private const double DeckFovMax  = 150.0;
    private const double ChatFovMin  = 60.0;   // tighter clamp for chat users
    private const double ChatFovMax  = 120.0;

    private const string YawVar   = "cam360_yaw";
    private const string PitchVar = "cam360_pitch";
    private const string FovVar   = "cam360_fov";
    private const string RollVar  = "cam360_roll";

    public bool Execute()
    {
        string camAction = "";
        if (args.ContainsKey("camAction") && args["camAction"] != null)
            camAction = args["camAction"].ToString().Trim().ToLowerInvariant();

        string command = "";
        if (args.ContainsKey("command") && args["command"] != null)
            command = args["command"].ToString().Trim().ToLowerInvariant();

        string input = "";
        if (args.ContainsKey("rawInput") && args["rawInput"] != null)
            input = args["rawInput"].ToString().Trim().ToLowerInvariant();

        // Deck / wrapper argument takes priority (trusted: full FOV range)
        if (!string.IsNullOrEmpty(camAction))
        {
            RunNamedMove(camAction, false);
            return true;
        }

        // No command = event trigger (e.g. Follow) -> spin
        if (string.IsNullOrEmpty(command))
        {
            DoSpin();
            return true;
        }

        if (command == "!spin")
        {
            DoSpin();
            return true;
        }

        if (command == "!barrelroll")
        {
            DoBarrelRoll();
            return true;
        }

        if (command == "!zoom")
        {
            if (input == "")
            {
                if (ChatFeedback)
                    CPH.SendMessage("Usage: !zoom 60-120, !zoom in, !zoom out, or !zoom reset");
                return true;
            }
            if (input == "in")         RunNamedMove("zoomin", true);
            else if (input == "out")   RunNamedMove("zoomout", true);
            else if (input == "reset") RunNamedMove("zoomreset", true);
            else                       RunNamedMove("zoom " + input, true);
            return true;
        }

        if (command == "!cam")
        {
            if (input == "")
            {
                if (ChatFeedback)
                    CPH.SendMessage("Usage: !cam front | back | left | right | up | down | level, !cam <degrees>, !cam tilt <angle>, !cam roll <angle>, !cam barrelroll, !cam zoom <fov>, !cam zoomin | zoomout | zoomreset");
                return true;
            }
            RunNamedMove(input, true);
            return true;
        }

        return true;
    }

    // ================= MOVE DISPATCH =================

    private void RunNamedMove(string move, bool fromChat)
    {
        string[] parts = move.Split(new[] { ' ' }, StringSplitOptions.RemoveEmptyEntries);
        string head = parts.Length > 0 ? parts[0] : "";
        string arg  = parts.Length > 1 ? parts[1] : "";

        double target;

        switch (head)
        {
            case "spin":       DoSpin();           return;
            case "barrelroll": DoBarrelRoll();     return;
            case "front":      MoveTo(0.0,   0.0); return;  // heading + level pitch/roll
            case "back":       MoveTo(180.0, 0.0); return;
            case "left":       MoveTo(-90.0, 0.0); return;
            case "right":      MoveTo(90.0,  0.0); return;
            case "up":         PitchOnly(90.0);    return;
            case "down":       PitchOnly(-90.0);   return;
            case "level":      LevelAll();         return;

            case "tilt":
                if (double.TryParse(arg, NumberStyles.Float, CultureInfo.InvariantCulture, out target))
                {
                    PitchOnly(Clamp90(target));
                }
                else if (fromChat && ChatFeedback)
                {
                    CPH.SendMessage("Usage: !cam tilt <-90 to 90>, e.g. !cam tilt 45");
                }
                return;

            case "roll":
                if (double.TryParse(arg, NumberStyles.Float, CultureInfo.InvariantCulture, out target))
                {
                    RollTo(Wrap180(target));
                }
                else if (fromChat && ChatFeedback)
                {
                    CPH.SendMessage("Usage: !cam roll <-180 to 180>, e.g. !cam roll 15 (or !cam barrelroll)");
                }
                return;

            case "zoomin":
                ZoomTo(GetStoredFov() - ZoomStep, fromChat);
                return;

            case "zoomout":
                ZoomTo(GetStoredFov() + ZoomStep, fromChat);
                return;

            case "zoomreset":
                ZoomTo(DefaultFov, false);
                return;

            case "zoom":
                if (double.TryParse(arg, NumberStyles.Float, CultureInfo.InvariantCulture, out target))
                {
                    ZoomTo(target, fromChat);
                }
                else if (fromChat && ChatFeedback)
                {
                    CPH.SendMessage("Usage: !cam zoom " + (int)ChatFovMin + "-" + (int)ChatFovMax + ", e.g. !cam zoom 80");
                }
                return;

            default:
                if (!double.TryParse(head, NumberStyles.Float, CultureInfo.InvariantCulture, out target))
                {
                    if (fromChat && ChatFeedback)
                        CPH.SendMessage("Couldn't read that. Try: !cam front | back | left | right | up | down | level, !cam 135, !cam tilt 45, !cam roll 15, or !cam zoom 90");
                    return;
                }
                YawOnly(target);
                return;
        }
    }

    // ================= CAMERA MOVES =================

    private void MoveTo(double targetYaw, double targetPitch)
    {
        double fromYaw   = GetStored(YawVar);
        double fromPitch = GetStored(PitchVar);
        double fromFov   = GetStoredFov();
        double fromRoll  = GetStored(RollVar);
        double yawDelta   = Wrap180(targetYaw - fromYaw);
        double pitchDelta = Clamp90(targetPitch) - fromPitch;
        double rollDelta  = Wrap180(0.0 - fromRoll);  // presets re-level roll too
        Animate(fromYaw, yawDelta, fromPitch, pitchDelta, fromFov, 0.0, fromRoll, rollDelta, PanDurationSec);
    }

    private void YawOnly(double targetYaw)
    {
        double fromYaw   = GetStored(YawVar);
        double fromPitch = GetStored(PitchVar);
        double fromFov   = GetStoredFov();
        double fromRoll  = GetStored(RollVar);
        double yawDelta  = Wrap180(targetYaw - fromYaw);
        Animate(fromYaw, yawDelta, fromPitch, 0.0, fromFov, 0.0, fromRoll, 0.0, PanDurationSec);
    }

    private void PitchOnly(double targetPitch)
    {
        double fromYaw    = GetStored(YawVar);
        double fromPitch  = GetStored(PitchVar);
        double fromFov    = GetStoredFov();
        double fromRoll   = GetStored(RollVar);
        double pitchDelta = Clamp90(targetPitch) - fromPitch;
        Animate(fromYaw, 0.0, fromPitch, pitchDelta, fromFov, 0.0, fromRoll, 0.0, PanDurationSec);
    }

    private void RollTo(double targetRoll)
    {
        double fromYaw   = GetStored(YawVar);
        double fromPitch = GetStored(PitchVar);
        double fromFov   = GetStoredFov();
        double fromRoll  = GetStored(RollVar);
        double rollDelta = Wrap180(targetRoll - fromRoll);
        Animate(fromYaw, 0.0, fromPitch, 0.0, fromFov, 0.0, fromRoll, rollDelta, PanDurationSec);
    }

    private void LevelAll()
    {
        double fromYaw   = GetStored(YawVar);
        double fromPitch = GetStored(PitchVar);
        double fromFov   = GetStoredFov();
        double fromRoll  = GetStored(RollVar);
        double pitchDelta = 0.0 - fromPitch;
        double rollDelta  = Wrap180(0.0 - fromRoll);
        Animate(fromYaw, 0.0, fromPitch, pitchDelta, fromFov, 0.0, fromRoll, rollDelta, PanDurationSec);
    }

    private void ZoomTo(double targetFov, bool fromChat)
    {
        double fromYaw   = GetStored(YawVar);
        double fromPitch = GetStored(PitchVar);
        double fromFov   = GetStoredFov();
        double fromRoll  = GetStored(RollVar);
        double clamped   = fromChat
            ? ClampRange(targetFov, ChatFovMin, ChatFovMax)
            : ClampRange(targetFov, DeckFovMin, DeckFovMax);
        double fovDelta  = clamped - fromFov;
        Animate(fromYaw, 0.0, fromPitch, 0.0, fromFov, fovDelta, fromRoll, 0.0, ZoomDurationSec);
    }

    private void DoSpin()
    {
        double fromYaw   = GetStored(YawVar);
        double fromPitch = GetStored(PitchVar);
        double fromFov   = GetStoredFov();
        double fromRoll  = GetStored(RollVar);
        Animate(fromYaw, 360.0, fromPitch, 0.0, fromFov, 0.0, fromRoll, 0.0, SpinDurationSec);
    }

    private void DoBarrelRoll()
    {
        double fromYaw   = GetStored(YawVar);
        double fromPitch = GetStored(PitchVar);
        double fromFov   = GetStoredFov();
        double fromRoll  = GetStored(RollVar);
        // Full 360 roll, ends exactly where it started
        Animate(fromYaw, 0.0, fromPitch, 0.0, fromFov, 0.0, fromRoll, 360.0, BarrelRollDurationSec);
    }

    private void Animate(double fromYaw, double yawDelta, double fromPitch, double pitchDelta,
                         double fromFov, double fovDelta, double fromRoll, double rollDelta,
                         double durationSec)
    {
        int totalSteps = Math.Max(1, (int)(durationSec * 1000.0 / StepMs));

        for (int i = 1; i <= totalSteps; i++)
        {
            double t = (double)i / totalSteps;
            double eased = t * t * (3.0 - 2.0 * t);
            SendAll(fromYaw + yawDelta * eased,
                    fromPitch + pitchDelta * eased,
                    fromFov + fovDelta * eased,
                    fromRoll + rollDelta * eased);
            Thread.Sleep(StepMs);
        }

        double finalYaw   = Wrap180(fromYaw + yawDelta);
        double finalPitch = Clamp90(fromPitch + pitchDelta);
        double finalFov   = ClampRange(fromFov + fovDelta, DeckFovMin, DeckFovMax);
        double finalRoll  = Wrap180(fromRoll + rollDelta);
        SendAll(finalYaw, finalPitch, finalFov, finalRoll);
        CPH.SetGlobalVar(YawVar,   finalYaw,   false);
        CPH.SetGlobalVar(PitchVar, finalPitch, false);
        CPH.SetGlobalVar(FovVar,   finalFov,   false);
        CPH.SetGlobalVar(RollVar,  finalRoll,  false);
    }

    // ================= OBS PLUMBING =================

    private void SendAll(double yaw, double pitch, double fov, double roll)
    {
        string json =
            "{\"sourceName\":\"" + Escape(SourceName) + "\"," +
            "\"filterName\":\"" + Escape(FilterName) + "\"," +
            "\"filterSettings\":{" +
            "\"yaw\":"   + Wrap180(yaw).ToString("F2", CultureInfo.InvariantCulture) + "," +
            "\"pitch\":" + Clamp90(pitch).ToString("F2", CultureInfo.InvariantCulture) + "," +
            "\"fov\":"   + ClampRange(fov, DeckFovMin, DeckFovMax).ToString("F2", CultureInfo.InvariantCulture) + "," +
            "\"roll\":"  + Wrap180(roll).ToString("F2", CultureInfo.InvariantCulture) +
            "}}";

        CPH.ObsSendRaw("SetSourceFilterSettings", json, ObsConnection);
    }

    private double GetStored(string varName)
    {
        try { return CPH.GetGlobalVar<double>(varName, false); }
        catch { return 0.0; }
    }

    // FOV needs a smarter fallback: an unset global reads as 0, which is not a valid FOV.
    private double GetStoredFov()
    {
        double f = GetStored(FovVar);
        if (f < DeckFovMin || f > DeckFovMax) return DefaultFov;
        return f;
    }

    // ================= HELPERS =================

    private static double ClampRange(double v, double min, double max)
    {
        return Math.Max(min, Math.Min(max, v));
    }

    private static double Wrap180(double degrees)
    {
        double d = degrees % 360.0;
        if (d > 180.0)  d -= 360.0;
        if (d < -180.0) d += 360.0;
        return d;
    }

    private static double Clamp90(double degrees)
    {
        return Math.Max(-90.0, Math.Min(90.0, degrees));
    }

    private static string Escape(string s)
    {
        return s.Replace("\\", "\\\\").Replace("\"", "\\\"");
    }
}
Part 5

Commands & triggers

  1. Commands tab → right-click → Add → create !cam. Set Location to Starts With — not Exact. Exact silently ignores !cam 90 because of the argument.
  2. Create !zoom the same way — Starts With (it takes arguments like !zoom 80).
  3. Create !spin and !barrelroll (Exact is fine — no arguments). Make sure all four commands are Enabled.
  4. In the Cam360 Control action, right-click the Triggers pane → Twitch → Chat → Command → select !cam. Repeat for !zoom, !spin, and !barrelroll.
  5. For the follower spin: right-click Triggers again → Twitch → Channel → Follow. The script treats any trigger without a chat command as a spin, so followers spin the camera automatically.

The direction presets (front/back/left/right) also return pitch and roll to level in one smooth move — one preset always rescues a camera chat has left crooked. Zoom holds across pans until changed or reset.

Tuning lives at the top of the script: PanDurationSec, SpinDurationSec, ZoomDurationSec, BarrelRollDurationSec, StepMs, ChatFeedback (false silences chat replies), plus the zoom block: DefaultFov (your resting FOV), ZoomStep, and the chat FOV clamp (ChatFovMin/ChatFovMax, default 60–120 so chat can’t mush-zoom or fisheye the stream). Deck buttons get the full 30–150 range.

Part 6

The queue — don't skip this

If a follow spin fires while a !cam move is mid-animation, both threads fight over the yaw value and the camera stutters. The fix takes 30 seconds:

  1. Settings → Action Queues → Add.
  2. Name it camera and leave Blocking checked (runs one action at a time).
  3. In the Cam360 Control action's settings, set its Queue to camera.

Stacked events now wait their turn instead of colliding. Any future camera actions (raid spins, sub moves) should go on this same queue.

Optional

Stream Deck / Decks buttons

Deck buttons trigger actions directly with no chat command attached, so pointing a deck button straight at Cam360 Control would always fall into the "no command = spin" path. The fix: keep the core action as the engine, and create one tiny wrapper action per button that passes it a camAction argument. The script above already understands this argument.

  1. Create a new action per button, e.g. Cam Left, Cam Right, Cam Front, Cam Back, Cam Spin.
  2. In each wrapper, add sub-action Core → Arguments → Set Argument — Name: camAction, Value: left (or right, front, back, up, down, level, spin, barrelroll, zoomin, zoomout, zoomreset, any number like 135, or two-word values like tilt 45, roll 15, zoom 90 — deck buttons get the full 30–150 FOV range).
  3. Add sub-action Core → Actions → Run Action → select Cam360 Control → leave "Run Immediately" UNCHECKED. Unchecked routes the move through the blocking camera queue, so mashing buttons queues moves politely instead of fighting over the yaw.
  4. In the Decks editor, point each button at its wrapper action. The wrappers need no triggers and no queue of their own.
Rule of thumb: triggers and code live on the core action; decks, hotkeys, and channel point redemptions point at wrappers. Never delete the core — everything routes through it.
Part 7

Test it

  1. From your own chat: !cam left, !cam back, !cam 135, !cam tilt 45, !zoom 80, !barrelroll, !cam front (should re-level but hold the zoom), !zoom reset, !spin.
  2. Test the follow trigger without a real follower: right-click the action → Test Trigger.
  3. Expected: smooth ~1.5-second eased pans always taking the shortest direction, ~1-second zooms, a 2.5-second barrel roll and a 4-second spin that both end exactly where they started.
Part 8

Getting the best image quality

A 360 camera spreads its pixels across an entire sphere, and this view only shows a slice of them — at ~95° FOV you're seeing roughly a quarter of the horizontal resolution stretched to full frame. It will never match a dedicated webcam, but these settings close most of the gap:

  1. Capture at full resolution. Uncheck "Use Preset" and set 2880×1440 manually (Part 2). This is the single biggest factor.
  2. Widen the FOV. Every degree wider uses more source pixels. 110–120° is noticeably sharper than 90°, with a slight fisheye look. Avoid narrow "zoomed" FOVs as a resting state.
  3. Lanczos scaling. Right-click the source → Scale Filtering → Lanczos. The default is soft.
  4. Mild sharpening. Add OBS's built-in Sharpen filter after the shader filter, around 0.08–0.15.
  5. Light the room well. Small 360 sensors get noisy in dim light, and noise plus upscaling reads as blur on stream.
  6. Feed the encoder. Spins are full-frame motion. Bump your bitrate or step your encoder preset up in quality — smearing during moves is usually bitrate starvation.
Pro layout: keep a normal cam (or the camera's single-lens mode) as your primary sharp "face" scene, and use the 360 chat-controlled view as its own scene or corner source. The 360 cam is the interactive toy — it doesn't have to carry your image quality.
Part 9

Troubleshooting

In the order things actually break:

Script won't compile — errors about dynamic binding or missing references

Add System.Core.dll in the References tab of the Execute C# Code window ("Add from Assembly Name"). This is the most common failure by a wide margin.

Commands do nothing, no errors anywhere

1) Check the !cam command's Location is Starts With, not Exact. 2) Check the command is Enabled and its trigger is actually attached to the action. 3) Check Stream Apps → OBS shows connected (green).

Compiles and triggers fire, but the camera doesn't move

SourceName or FilterName in the script doesn't exactly match OBS. Copy-paste the names from OBS directly into the script constants — one wrong space breaks it silently.

Changed DefaultFov (or another default) but OBS snaps back to the old value

The script remembers its last position in Streamer.bot global variables (cam360_yaw, cam360_pitch, cam360_fov, cam360_roll) and those win over the constants. After changing DefaultFov, run !zoom reset once to store the new value — or just restart Streamer.bot, which clears the globals so the first move falls back to your new defaults. Same logic applies after moving the sliders by hand in OBS: the script doesn’t see manual slider changes and will snap back on the next move.

Camera jumps or stutters when events overlap

You skipped Part 6. Set up the blocking camera queue and assign the action to it.

Visible seam or heavy distortion in the view

Thin vertical line exactly when facing the rear (yaw 180): you're running the original (v1) shader. The v2.1 shader in this guide fixes it with a horizontal-wrap sampler — re-copy the shader above into your .shader file and reload the filter. Ghosting or misalignment on close objects at the sides (±90°): that's the camera's optical stitch line where the two lenses meet, worst on close subjects. Fix is physical: rotate the camera so the lenses face what matters and the stitch zones land somewhere boring. Everything looks warped: confirm the camera is outputting equirectangular (a 2:1 resolution like 2880×1440), not a fisheye or already-reframed mode.

Chat commands animate from the wrong starting position

The script tracks yaw itself (starting at 0). If you drag the OBS slider by hand, the next chat command animates from the script's last known angle, not the slider's. Run any !cam command and it re-syncs from there.

OBS WebSocket connection refused

Confirm OBS WebSocket is enabled on port 4455 and the password in Streamer.bot matches. If another app is squatting on the port, change it in both places.
